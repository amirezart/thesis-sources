# آرشیوِ پایان‌نامه

پوشهٔ کاریِ آرشیو. همه‌چیز متنِ ساده است تا در وی‌اس کد باز شود و در گیت دیف بگیرد.

## ساختار

```
archive/
├─ README.md                      ← همین فایل
├─ works-index.csv                ← آثارِ خودم
├─ works-index-README.md
├─ references-index.csv           ← آثارِ دیگران (ارجاع و الهام)
├─ references-index-README.md     ← راهنمای ستون‌ها و الگوهای استنادِ APA
├─ inspiration-map.csv            ← رابطهٔ میانِ آثارِ خودم و آثارِ مرجع
├─ reference-note-template.md     ← الگوی یادداشتِ تحلیلی
├─ references/                    ← تصاویر و پی‌دی‌افِ بک‌آپِ آثارِ مرجع
└─ notes/                         ← یادداشت‌های تحلیلیِ هر اثر
```

## قاعده‌ها

**سه فایلِ سی‌اس‌وی منبعِ حقیقت‌اند.** هرچیزِ دیگری از روی آن‌ها ساخته می‌شود، نه برعکس.

**کدها هرگز بازاستفاده نمی‌شوند.** `CC-##` برای آثارِ خودم، `RF-###` برای آثارِ مرجع. اثرِ کنارگذاشته‌شده پاک نمی‌شود؛ در ستونِ یادداشت علامت می‌خورد.

**هر اثری که لینک دارد ولی بک‌آپ ندارد، آرشیو نشده است.** لینک می‌شکند. فایلِ بک‌آپ در `references/` با همان کدِ اثر نام‌گذاری می‌شود، مثلاً `RF-001_p5live.pdf`.

**یادداشتِ تحلیلی فقط برای آثارِ «مصداق تحلیلی».** برای «ارجاع نظری» و «الهام عملی» همان ردیفِ سی‌اس‌وی کافی است. نامِ فایل: `notes/RF-001.md`.

## وضعیتِ فعلی

- آثارِ مرجع: ۱۰ ردیف ثبت‌شده (تد دیویس، دانیل ونتسل، استودیو دیا، پاتریک هوبنر)
- آثارِ خودم: ۳ ردیفِ خالیِ نمونه
- بک‌آپِ محلی: یک بک‌آپ متنی برای `RF-010` ثبت شده؛ بک‌آپ بصری و ویدئویی آن هنوز کامل نیست و سایر آثار نیز فاقد بک‌آپ‌اند
- ستونِ «زیربخش مرتبط»: همه خالی — این را فقط خودم می‌توانم پر کنم
